# Figma UI/UX Design – Trading Platform

## Overview

This folder contains the **Figma UI/UX designs for the Trading Platform web application**. The designs were created using **Figma** to plan the user interface, navigation structure, and user interaction flow before implementing the frontend.

The design process includes **complete website wireframes and high-fidelity UI designs**, ensuring a clear visual representation of the platform.

---

## Wireframes

As the first step of the design process, **wireframes were created for the entire website**. These wireframes represent the basic layout and structure of each page without detailed visual styling.

The purpose of creating wireframes was to:

* Plan the **overall layout of the website**
* Define **navigation flow between pages**
* Organize **information hierarchy**
* Decide placement of UI elements
* Ensure design consistency across the platform

The wireframes cover all major pages of the system, providing a complete structural overview of the platform.

---

## UI Design

After finalizing the wireframes, **high-fidelity UI designs were created in Figma**. These designs include visual styling such as colors, typography, spacing, and component layouts.

The UI design focuses on:

* Clear and intuitive user interface
* Consistent design components
* Organized dashboard layout
* Structured trading interface
* Clean navigation and user interaction

The design is optimized for **desktop resolution (1440px width)** to maintain proper alignment and spacing of elements.

---

## Pages Designed

The following pages were designed in the Figma project:

* Landing Page
* User Registration Page
* Login Page
* User Dashboard
* Trading Interface
* Portfolio Page
* Watchlist Page
* Market Data / Instruments Page
* Courses and Learning Modules
* Challenges and Quiz Pages
* Notifications Page

These designs collectively represent the **complete workflow of the trading platform**.

---

## Prototype

Figma's prototyping feature was used to create **interactive navigation between pages**. This allows simulation of the user experience before the development phase.

Examples include:

* Navigation from the landing page to login and registration pages
* Dashboard navigation between trading, portfolio, and watchlist pages
* Access to learning modules and challenges
